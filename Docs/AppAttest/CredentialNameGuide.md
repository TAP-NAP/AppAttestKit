## What It Is

`credentialName` is a caller-defined string that names one App Attest
credential. AppAttestKit uses it only as a local metadata lookup key:

```text
credentialName -> keyId / credentialId / status / environment / timestamps
```

It is not an Apple App Attest claim, and it is not generated by the kit.

## What AppAttestKit Stores

The Keychain store saves the Apple App Attest `keyId` handle and backend
credential metadata for the credential name. It does not store the App Attest
private key. Apple manages the private key material.

## Naming Defaults

The example app uses:

```swift
let credentialName = "installation_keyid"
```

Apps may choose any stable, non-empty string that makes sense for their backend
contract. Changing the string means using a different App Attest credential.

## Responsibilities

AppAttestKit:

- Normalizes and validates the non-empty credential name.
- Stores and retrieves metadata for that exact name.
- Runs attestation and assertion with the stored Apple `keyId`.

Caller and backend:

- Choose the credential naming scheme.
- Decide which business operation should use which credential name.
- Avoid putting sensitive raw data in the credential name.
